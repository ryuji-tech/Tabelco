
# 制作アプリのタイトル
店舗検証フォーム（PHP / CSV）

## 制作アプリの説明

子ども連れで行ける飲食店の情報を、運営が現地で確認して登録・一覧表示するための
Webフォームです。PHPの基礎（フォーム送信・CSVへの保存・読み込み表示）を題材に実装しました。

## 制作アプリの説明（機能）

- **入力フォーム（`index.php`）** … 店舗の基本情報と、現地確認した4つの視点
  （食器まわり／授乳スペース／ベビーカー／おむつ替え）を 0〜2 で記録する。
- **保存（`create.php`）** … 送信内容を `data/shops.csv` に1行ずつ追記する。
- **一覧（`read.php`）** … CSVを読み込み、カード形式で表示する。
  4視点の 0/1/2 は ○△× のバッジに変換して表示。

## 工夫した点、こだわった点（実装ポイント）

- `$_POST['キー']` でフォーム入力を受け取り、`fputcsv()` / `fgetcsv()` で
  CSVの読み書きを行う（カンマ・改行を自動でエスケープ）。
- `htmlspecialchars()` で出力をエスケープし、XSS・表示崩れを防ぐ。
- 複数選択（時間帯・設備・食器アイテム）は `name="xxx[]"` で配列受け取り、
  `implode('|', …)` で保存／`explode('|', …)` で表示に展開する。
- 沿線を選ぶと最寄駅が連動して切り替わる（PHPの駅データを `json_encode` で
  JavaScriptに渡し、`change` で選択肢を作り直す）。
- 最寄駅から所属エリア（ブロック）を連想配列で自動算出する。
- 一覧の写真はサムネイル表示（`object-fit: cover` ＋ 読み込み失敗時のフォールバック）。

## 難しかった点・次回トライしたいこと（または機能）

初め地域などのデータは手打ちする形式でしたが、データの分類を後々行なう際にスマートになるように地下鉄などの駅名から地域データの名称に流用し、選択できる形式にしました。よく入力しそうな項目は選択形式で入力を完了できるようにしました。住所入力などは将来的にAPIを用いて半自動化を検討。電話や営業時間などまだまだ収集すべき情報が多く、別途フォームを用意する予定。

## 感想

このフォームのカテゴリの中に住所を入れると、前述のように電話番号など足りていない要素もあるので少し浮く状態なのだが、最小限の情報で進捗させる際に地図を掲載するための住所の情報は必要なので今回のフォームに意図的に入れ込んだ。まとめてフォームに詰め込んでも良かったのであるが、認知負荷が膨大になることと、後からしか収集できない情報の項目もあったので、あえてフォームを分割してみることにした。項目データの要素を後で追加することは可能なのであるが、実際の体感としては掲載をフォームの箱を作っても、収集したいデータをきちんと決めておかないと、フォーマットを修正するたびに過去にCSVで出力したデータとの連携に支障が出てしまうことがよくわかった。モリモリも良くないが、安易に後で項目を追加しようと考えるのも良くないなと感じた。今回取ったフォーム分割の手法によって項目修正時の影響を軽微にできる可能性もあるのではないかとプラスに考えてみることにした。

## ファイル構成

| ファイル | 役割 |
|---|---|
| `index.php` | 入力画面（店舗検証フォーム） |
| `create.php` | 保存処理（CSVへ追記） |
| `read.php` | 一覧表示 |
| `styles.css` | スタイル |
| `data/` | `shops.csv` が自動生成される場所（初期は空） |

データの流れ：`index.php`（入力）→ `create.php`（保存）→ `read.php`（表示）

## 動作環境・実行方法

- PHP 8 系（XAMPP などの Apache + PHP 環境）。
- `tabelco_php1` を `htdocs` 配下に置き、ブラウザで
  `http://localhost/tabelco_php1/index.php` を開く。
- PHPファイルは直接開かず、必ず `http://localhost/...` 経由でアクセスすること。
- `data/` フォルダはCSVを書き込むため、書き込み権限が必要。

## 備考

- `data/*.csv` は実データのため Git 管理から除外している（`.gitignore`）。
